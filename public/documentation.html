MovieFetcher API

Welcome to my MovieFetcher API documentation!

I am creating a movieAPI for my application called "MovieFetcher" that interacts with a database that stores data about different movies. The web application provides users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.

Getting Started

How to get or change data about movies, genres and directors?

Get the list of data about all movies GET /movies None A JSON object holding data about all movies.

Get the data about a genre by name GET /movies/genre/:genreName None A JSON object holding data about a one specific genre:
{
"name": "Action",
"description": "The action film is a film genre that [...]."
}
Get the all data about a director by name GET /movies/director/:directorName None A JSON object holding data about a one specific director:
{
"name": "Mike Newell",
"bio": "Michael Cormac \"Mike\" Newell is an English director and producer [...].",
"birth": "28 Mar 1942",
"death": null
}
Get the all movies in which was a certain actor GET /movies/:actorName None A JSON object holding data about a movie/s in which was a certain actor.
Get the all movies which were released in a certain year GET /movies/:releaseYear None A JSON object holding data about movies released in a certain year.
Get the all movies with a certain rating GET /movies/:rating None A JSON object holding data about movies with a certain rating.
Add new actor to the actor's array in a certain movie POST /movies/:id/:actorName None A text message indicating whether the new actor has successfully added to certain movie.
Delete a certain movie DELETE /movies/:id None A text message indicating whether the movie has successfully removed.

How to manage data about users?

BUSINESS LOGIC: HTTP METHOD: ENDPOINT: REQUEST BODY DATA FORMAT: RESPONSE BODY DATA FORMAT:
Add (register) new users POST /users A JSON object holding data about the new user to add, structured like the following example:
{
"username": "Olga White",
"password": "passwordExample66",
"email": "olgawhite66@example.com",
"favoriteMovies": []
} 		A JSON object holding data about the new user that was added:
{
"username": "OlgaWhite",
"password": "passwordExample66",
"email": "olgawhite66@example.com",
"favoriteMovies": [],
"id": "d1dccc24-1151-452c-bcc3-29698835da3d"
}
Update (change) user data PUT /users/:id A JSON object holding new username that should be changed:
{
"name": "Leandro Castillo"
} 		A text message indicating whether the username was successfully changed.
Add a movie to the user's favorites list POST /users/:id/:newFavMovie None A text message indicating whether the user has successfully added a movie to the favorites list.
Delete a movie from user's list of favorites DELETE /users/:id/:newFavMovie None A text message indicating whether the user has successfully removed a movie from the favorites list.
Delete user's account (deregester) DELETE /users/:id None A text message indicating whether the user's account has successfully deleted.
